# Section 6: Wazuh SIEM Setup

Deploy Wazuh 4.7.5 on siem01 (Ubuntu 24.04 LTS).

## 6.1 — Install Wazuh Server

**Script:** [scripts/bash/install-wazuh-server.sh](../scripts/bash/install-wazuh-server.sh)

> **Important:** Wazuh 4.7.5 does not officially support Ubuntu 24.04. Use the `-i` (ignore check) flag to bypass the OS compatibility check. The installation works correctly despite the warning.

```bash
# Download Wazuh install script
curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh

# Run with -i to bypass Ubuntu 24.04 OS check
sudo bash wazuh-install.sh -a -i
```

Installation takes ~10 minutes. At the end, the script outputs:

```
INFO: --- Summary ---
INFO: You can access the web interface https://192.168.56.103
    User: admin
    Password: <auto-generated>
```

**Save this password.** You will use it to log in to the Wazuh dashboard.

## 6.2 — Access the Wazuh Dashboard

1. Open a browser on your host machine
2. Navigate to `https://192.168.56.103`
3. Accept the self-signed certificate warning
4. Log in:
   - **Username:** `admin`
   - **Password:** `<YOUR_WAZUH_PASSWORD>` (from install output above)

## 6.3 — Install Wazuh Agents on Windows VMs

**Script:** [scripts/powershell/11-install-wazuh-agent.ps1](../scripts/powershell/11-install-wazuh-agent.ps1)

Run on **dc01**, **dc02**, and **wkstn01** after the Wazuh dashboard is reachable:

```powershell
# Ensure temp directory exists
if (-not (Test-Path 'C:\Temp')) { New-Item -ItemType Directory -Path 'C:\Temp' | Out-Null }

# Download agent installer
Invoke-WebRequest -Uri 'https://packages.wazuh.com/4.x/windows/wazuh-agent-4.7.5-1.msi' `
    -OutFile 'C:\Temp\wazuh-agent.msi'

# Install and enroll agent
msiexec.exe /i 'C:\Temp\wazuh-agent.msi' /q `
    WAZUH_MANAGER='192.168.56.103' `
    WAZUH_REGISTRATION_SERVER='192.168.56.103' `
    WAZUH_AGENT_GROUP='windows'

# Start the agent service
NET START WazuhSvc
```

If you prefer, run the tracked script directly instead:

```powershell
powershell -ExecutionPolicy Bypass -File .\11-install-wazuh-agent.ps1
```

## 6.4 — Verify Agent Enrollment

On siem01:

```bash
sudo /var/ossec/bin/agent_control -l
```

Expected: dc01, dc02, wkstn01 listed as `Active`.

In the Wazuh Dashboard: **Agents** → all three Windows VMs should show **Active** status.

## Service Management (siem01)

```bash
# Check Wazuh status
sudo systemctl status wazuh-manager

# Restart after config changes
sudo systemctl restart wazuh-manager

# View manager logs
sudo tail -f /var/ossec/logs/ossec.log
```
