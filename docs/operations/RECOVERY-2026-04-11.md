### 1c — If agent files are missing (deleted during storage cleanup)

First, confirm whether the agent directory still exists:

```powershell
VBoxManage guestcontrol "dc01" run `
  --exe "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" `
  --username "Administrator" --password "LabAdmin@2026!" `
  -- -NonInteractive -Command "Test-Path 'C:\Program Files (x86)\ossec-agent\ossec.conf'" `
  --wait-stdout --wait-stderr
```

If output is `False`, the agent config is gone. Check for the Wazuh MSI on the host:

```
C:\Users\demar\AppData\Local\Temp\wazuh-agent-4.7.5-1.msi
```

If the MSI is present, reinstall via scheduled task (Session 0 workaround — same method
used during initial enrollment in Session 2):

```powershell
# Step 1: Write password file (avoids ! escaping)
Set-Content -Path "C:\Users\demar\tmp_wazuh_pass.txt" -Value "LabAdmin@2026!"

# Step 2: Create scheduled task to run MSI install as SYSTEM
VBoxManage guestcontrol "dc01" run `
  --exe "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" `
  --username "Administrator" --password "LabAdmin@2026!" `
  -- -NonInteractive -Command @"
    schtasks /create /tn WazuhReinstall /tr 'msiexec /i \"C:\wazuh-agent-4.7.5-1.msi\" /quiet WAZUH_MANAGER=192.168.56.50 WAZUH_AGENT_NAME=DC01' /sc once /st 00:00 /ru SYSTEM /f
"@ `
  --wait-stdout --wait-stderr

# Step 3: Copy MSI to dc01 guest first
VBoxManage guestcontrol "dc01" copyto `
  "C:\Users\demar\AppData\Local\Temp\wazuh-agent-4.7.5-1.msi" `
  "C:\wazuh-agent-4.7.5-1.msi" `
  --username "Administrator" --password "LabAdmin@2026!"

# Step 4: Run the scheduled task
VBoxManage guestcontrol "dc01" run `
  --exe "C:\Windows\System32\cmd.exe" `
  --username "Administrator" --password "LabAdmin@2026!" `
  -- /c "schtasks /run /tn WazuhReinstall" `
  --wait-stdout --wait-stderr

# Step 5: Wait 60 seconds then set service to auto-start
Start-Sleep -Seconds 60
VBoxManage guestcontrol "dc01" run `
  --exe "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" `
  --username "Administrator" --password "LabAdmin@2026!" `
  -- -NonInteractive -Command "Set-Service -Name 'Wazuh' -StartupType Automatic; Start-Service 'Wazuh'" `
  --wait-stdout --wait-stderr
```

## Step 2 — Validate AD Replication on dc02 (P1 — BLOCKING)

dc02 was restored to `session7-complete-2026-04-08`. Confirm it is replicating correctly
with dc01 before treating the environment as healthy.

### 2a — Run repadmin replsummary on dc02

```powershell
VBoxManage guestcontrol "dc02" run `
  --exe "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" `
  --username "Administrator" --password "LabAdmin@2026!" `
  -- -NonInteractive -Command "repadmin /replsummary" `
  --wait-stdout --wait-stderr
```

**Healthy output looks like:**
```
Replication Summary Start Time: ...
Beginning data collection for replication summary, this may take awhile:
Source DSA          largest delta    fails/total %%   error
 DC01               00m:00s            0 /   5    0
Destination DSA     largest delta    fails/total %%   error
 DC01               00m:00s            0 /   5    0
```

Zero fails = replication is healthy.

**If replication is failing:**

```powershell
# Check detailed replication status
VBoxManage guestcontrol "dc02" run `
  --exe "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" `
  --username "Administrator" --password "LabAdmin@2026!" `
  -- -NonInteractive -Command "repadmin /showrepl dc02" `
  --wait-stdout --wait-stderr

# Force a sync attempt
VBoxManage guestcontrol "dc02" run `
  --exe "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" `
  --username "Administrator" --password "LabAdmin@2026!" `
  -- -NonInteractive -Command "repadmin /syncall /AdeP" `
  --wait-stdout --wait-stderr
```

> **HARD STOP:** If replication errors persist after syncall, stop here and diagnose
> manually via dc02 console before taking any further automated action. Common causes:
> firewall blocking 445/389/3268, time skew > 5 minutes, or SRV record drift.

### 2b — Confirm dc01 sees dc02 as a replication partner


Look for `dc02` as a source DSA with `Last attempt` showing a recent successful time.
